# Accessibility

Design inclusively and meet legal standards — WCAG 2.2, screen reader UX, keyboard navigation, ARIA patterns, and German BFSG compliance.

**7 skills** and **3 commands** for accessible design in [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

## Skills

| Skill | Description |
|-------|-------------|
| [wcag-compliance](./skills/wcag-compliance) | Apply WCAG 2.1/2.2 success criteria at AA and AAA levels. |
| [color-contrast](./skills/color-contrast) | Ensure accessible contrast ratios across all UI states. |
| [screen-reader-ux](./skills/screen-reader-ux) | Design semantic structure and reading order for screen readers. |
| [keyboard-navigation](./skills/keyboard-navigation) | Design keyboard-first interaction patterns. |
| [focus-management](./skills/focus-management) | Define focus states, focus traps, and skip navigation. |
| [aria-patterns](./skills/aria-patterns) | Apply ARIA roles, labels, and live regions correctly. |
| [bfsg-compliance](./skills/bfsg-compliance) | Meet German BFSG (Barrierefreiheitsstärkungsgesetz) requirements. |

## Commands

| Command | Description |
|---------|-------------|
| `/accessibility:accessibility-audit` | Run a full accessibility audit across a product or component. |
| `/accessibility:wcag-review` | Review designs against WCAG 2.2 AA criteria. |
| `/accessibility:inclusive-review` | Review designs across disability types for inclusive coverage. |
