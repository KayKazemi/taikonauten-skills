# Color Identity

Build a brand color system that is expressive, flexible, and accessible.

## What You Do

You help define brand colors not just as a palette, but as a system with meaning, hierarchy, and rules for use across every brand touchpoint.

## Color System Structure

- **Primary brand color**: The dominant, signature color — most associated with the brand
- **Secondary colors**: Supporting palette that expands expressiveness without competing
- **Neutral colors**: Grays and off-whites for text, backgrounds, and UI surfaces
- **Accent/highlight colors**: High-energy colors used sparingly for emphasis
- **Semantic colors**: Functional roles — success, warning, error, info (relevant for digital products)

## Color Attributes to Define

For each color: HEX, RGB, CMYK, Pantone (if print is relevant), HSL
For digital: include CSS custom property name and token name

## Brand Color Principles

- **Personality expression**: Each color choice should connect to a brand value or personality trait
- **Hierarchy**: The palette has a clear dominant/supporting/accent structure
- **Flexibility**: Works on light and dark backgrounds
- **Accessibility**: All text/background combinations meet WCAG 2.2 AA (4.5:1 for body text, 3:1 for large text)
- **Distinctiveness**: The palette reads as ownable — not generic to the industry

## Usage Guidance

- Define usage ratios (e.g., 60% primary, 30% secondary, 10% accent)
- Specify approved color combinations and prohibited combinations
- Show the palette in use across digital and print contexts
