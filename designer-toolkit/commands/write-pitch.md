# /write-pitch

Build a complete pitch deck end-to-end.

## Steps

1. **Audience and goal**: Define who will be in the room, what they care about, and what a successful outcome looks like for $ARGUMENTS.
2. **Positioning**: Use the `pitch-deck` skill to identify the hook — the one insight or provocation that will earn the audience's attention from slide one.
3. **Story structure**: Use the `pitch-deck` skill to map the full narrative arc — problem, solution, proof, team, ask.
4. **Proof selection**: Use the `case-study` skill to select and shape 1–2 case studies that map most directly to the audience's challenge.
5. **Slide writing**: Use `ux-writing` skill principles to write clear, punchy headlines for each slide. Every headline should be a statement, not a label.
6. **Rationale layer**: Use the `design-rationale` skill to sharpen any sections where design decisions need to be explained and defended.
7. **Review**: Check each slide passes the "so what?" test — if you can't articulate why this slide moves the audience toward yes, cut or reframe it.

## Output

Complete pitch deck outline: slide-by-slide with headline, content notes, visual direction, and speaker notes.

Consider following up with `/designer-toolkit:build-presentation` to develop the full slide content, or `/client-pitch:write-proposal` if a written proposal needs to follow the pitch.
