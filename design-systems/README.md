# Design Systems

Build, document, and maintain scalable design systems — from tokens and components to accessibility and theming.

**8 skills** and **3 commands** for design systems in [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

## Skills

| Skill | Description |
|-------|-------------|
| [design-token](./skills/design-token) | Define and organise design tokens (color, spacing, typography, elevation) with naming conventions and usage guidance. |
| [component-spec](./skills/component-spec) | Write a detailed component specification including props, states, variants, and accessibility. |
| [pattern-library](./skills/pattern-library) | Structure a pattern library entry with problem context, solution, and examples. |
| [naming-convention](./skills/naming-convention) | Establish naming convention systems for design elements, components, and tokens. |
| [accessibility-audit](./skills/accessibility-audit) | Conduct an accessibility audit against WCAG guidelines with severity ratings. |
| [theming-system](./skills/theming-system) | Design a theming architecture for brand variants, dark mode, and high-contrast. |
| [icon-system](./skills/icon-system) | Create an icon system specification covering grid, sizing, naming, and categories. |
| [documentation-template](./skills/documentation-template) | Generate structured documentation templates for design system artefacts. |

## Commands

| Command | Description |
|---------|-------------|
| `/design-systems:audit-system` | Audit a design system for consistency, completeness, and accessibility. |
| `/design-systems:create-component` | Scaffold a full component specification from a name or description. |
| `/design-systems:tokenize` | Extract and organise design tokens from an existing design or stylesheet. |
